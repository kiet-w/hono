# Project Instructions

## Git Workflow
- **Branching**: Never commit directly to `main`. Always create a new feature branch for any changes or new tasks.
- **Review**: Use branches to allow the user to review changes and learn from the implementation.

## Shell Commands
- **Windows Compatibility**: Avoid running `.sh` files directly if they trigger the "Select an app" dialog. Use PowerShell-compatible commands or execute scripts via an explicit interpreter (e.g., `sh script.sh`).
